<h1>API DOCUMENTATION</h1>

<span>
The Emporio Analytics assignment API is organised around REST. Our API has predictable resource-oriented URLs, accepts form-encoded request bodies, returns JSON-encoded responses, and uses standard HTTP response codes authentication.
</span>

<br/>
<br/>

<h3>GET - <span>/products/getMake</span></h3>  

    fields : [
        {
            id: String,
            name: String,
            description: String,
            categoryGroupId: String,
            files: Array <-- images,
            category: Array,
            product: Array,
        },
        {...},
        {...},
    ]

Retrieve all the makes from the inventory list

<br/>

<h3>GET - <span>/products/specificVariantExterior/{id}</span></h3>  

    fields : {
        String : {
            id: String,
            name: String,
            groupName: String,
            objects: [
                {
                    id: String
                    name: String,
                    price: String,
                    isDefault: String,
                    productId: String,
                    productVariantId: String,
                    files : Array <-- images
                }
            ]
        },
        {...},
        {...},
    }

Retrieve dynamic keys and values of the product variant exterior, remember to use dynamic mapping to extract the keys and values, hard coding may produce errors in frontend as admin may change the variants anytime

<h3>GET - <span>/products/specificGradeProductOption/{id}</span></h3>

                id: String,
                name: String,
                price	Number,
                isDefault	Boolean,
                editable	Boolean,

                files: Array

        ],
        [...],
        [...],
    }
